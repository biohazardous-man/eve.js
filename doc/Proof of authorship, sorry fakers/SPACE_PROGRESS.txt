EvEJS Space Completion Checklist
Updated: 2026-03-10

Scope
- This document now tracks the implemented station-to-space slice, not the older planning-only state.
- The original undock plan phases are complete for the intended scope:
  - undock,
  - ballpark bootstrap,
  - destiny movement,
  - docking back,
  - multi-player visibility,
  - warp,
  - stargate jump,
  - unified workspace-local reference data.

Database and data-source status
- DONE: mutable gameplay state remains under `server/src/newDatabase`.
- DONE: static universe reference data is also available under `server/src/newDatabase/data`:
  - `solarSystems`
  - `stations`
  - `celestials`
  - `stargates`
  - `movementAttributes`
  - `shipTypes`
  - `skillTypes`
- DONE: runtime readers use the shared reference-data loader and no longer read `server/data/sde` at runtime.
- DONE: runtime code does not reference any path outside this workspace.
- DONE: build scripts generate/import workspace-owned data into the `newDatabase` table format.

Runtime and ballpark status
- DONE: `server/src/space/worldData.js`
  - loads systems, stations, celestials, stargates, and movement attributes from `newDatabase`
  - supports all-system lazy lookup, not Jita-only hardcoding
- DONE: `server/src/space/runtime.js`
  - owns solar-system scenes
  - owns static and dynamic entities
  - tracks session attachment and removal
  - persists ship `spaceState`
  - broadcasts enter/leave visibility changes
- DONE: dynamic ballpark bootstrap
  - `AddBalls2`
  - `SetState`
  - immediate mode follow-up updates
  - removal propagation via `RemoveBalls`
- DONE: multi-player visibility
  - two in-space players in the same system see each other enter, move, warp, and leave

  - `AddBalls2`

- DONE: implemented ship modes
  - `STOP`

Session and transition status
- DONE: undock transition in `server/src/space/transitions.js`
  - validates docked state
  - resolves station and destination system
  - moves ship item into space
  - updates character record
  - applies in-space session values
  - emits `OnCharNoLongerInStation`
  - attaches to ballpark runtime
- DONE: docking transition
  - detaches from space
  - moves ship back to station
  - restores station session state
- DONE: disconnect/reconnect support
  - stale dynamic entities are removed on detach
  - in-space characters can be restored into runtime from persisted ship `spaceState`
- DONE: stargate jump transition
  - validates gate targeting/proximity
  - moves ship to destination system
  - reapplies session state
  - rebinds/reboots the new destination ballpark
